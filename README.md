# Прогнозирование стоимости страховых выплат

## Описание проекта
Цель проекта — разработать модель машинного обучения для предсказания
итоговой стоимости страхового возмещения (`UltimateIncurredClaimCost`) на
основе характеристик работника и параметров страхового случая.

Результаты оформлены как многостраничное Streamlit-приложение
(`st.navigation` + `st.Page`):

- **Анализ и модель** — загрузка данных, EDA, предобработка, обучение
  четырёх моделей регрессии (Linear, Ridge, Random Forest, XGBoost),
  сравнение метрик MAE / MSE / RMSE / R², график важности признаков и
  интерактивная форма предсказания.
- **Презентация** — слайды на `streamlit-reveal-slides` с описанием
  бизнес-задачи, этапов работы и результатов.

## Датасет
Используется датасет **Workers Compensation** (OpenML id = `42876`),
содержащий 100 000 записей о страховых случаях:

- Категориальные признаки: `Gender`, `MaritalStatus`, `PartTimeFullTime`,
  `ClaimDescription`.
- Datetime-признаки: `DateTimeOfAccident`, `DateReported` (преобразуются
  в `AccidentMonth`, `AccidentDayOfWeek`, `ReportingDelay`).
- Числовые: `Age`, `DependentChildren`, `DependentsOther`, `WeeklyPay`,
  `HoursWorkedPerWeek`, `DaysWorkedPerWeek`, `InitialCaseEstimate`.
- Целевая переменная: `UltimateIncurredClaimCost`.

При первом запуске датасет автоматически скачивается через
`sklearn.datasets.fetch_openml(data_id=42876)` и сохраняется в
`data/workers_compensation.csv`, чтобы повторные запуски были мгновенными.

## Установка и запуск

> **Важно: рекомендуемая версия Python — 3.10 или 3.11.**
> Зафиксированные в `requirements.txt` версии `pandas==1.5.3`,
> `numpy==1.24.0`, `scikit-learn==1.2.0` и т.д. соответствуют спецификации
> задания и официально не поддерживают Python 3.12+.

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Kaztat37/workers_compensation_project.git
   cd workers_compensation_project
   ```

2. (Рекомендуется) создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv .venv
   source .venv/bin/activate    # Linux / macOS
   .venv\Scripts\activate       # Windows
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Запустите приложение:
   ```bash
   streamlit run app.py
   ```

5. Откройте `http://localhost:8501` в браузере.

## Как пользоваться

1. На странице **Анализ и модель** в боковой панели выберите размер выборки
   (по умолчанию 30 000 — компромисс скорости и качества), долю тестовой
   выборки (по умолчанию 20%) и решите, обучать ли модели на
   `log1p(UltimateIncurredClaimCost)` (по умолчанию — да; критично для
   сильно скошенной целевой переменной), затем нажмите **«Загрузить данные
   и обучить модели»**.
2. Дождитесь обучения четырёх моделей. На странице появятся:
   - таблица сравнения метрик (MAE, MSE, RMSE, R², R² на log-шкале);
   - график «Предсказания vs реальные значения» для выбранной модели;
   - горизонтальная диаграмма важности признаков (Random Forest);
   - распределение целевой переменной (лог. шкала).
3. Спуститесь к форме «Предсказание для нового случая», заполните параметры
   работника и страхового случая и получите оценку
   `UltimateIncurredClaimCost`.
4. На странице **Презентация** в сайдбаре доступны переключатели темы,
   высоты слайдов, перехода и плагинов Reveal.js — после изменения слайды
   автоматически переинициализируются.

## Метрики качества
Модели сравниваются по четырём метрикам:

- **MAE** — средняя абсолютная ошибка (доллары).
- **MSE** — средняя квадратичная ошибка.
- **RMSE** — корень из MSE (доллары).
- **R²** — коэффициент детерминации.

На выборке 30 000 записей лучший результат, как правило, показывают
**Random Forest** и **XGBoost** (R² на log-шкале ≈ 0.4).

## Воспроизводимость
- `random_state=42` фиксируется во всех моделях и в `train_test_split`.
- Результаты обучения кешируются через `st.cache_resource`, а данные —
  через `st.cache_data`, чтобы повторные действия не пересчитывались.
